## 第4页：创新思路 —— 从"信息查询"到"智能服务"

**标题**：不只是问答，更是能动手办事的AI Agent

**要点**：
- **核心理念**：让AI不仅能"回答"，更能"执行"——通过自然语言直接完成校园事务
- **技术选型**：ReAct（Reasoning + Acting）架构，支持多步推理与工具调用
- **本地化执行**：Agent直接调用本地工具（查课表、搜空教室、读文件），无需依赖外部API
- **深度集成**：与哈工大教务系统（EAS）深度对接，支持深圳/本部/威海三校区

## 第5页：产品架构 —— 端侧智能的技术底座

**标题**：端侧AI + 教务中台 + 知识后端，构建可信校园服务

**要点**：
- **客户端**：Android原生应用（Kotlin），Hilt依赖注入，Room本地数据库；iOS版本TestFlight内测中
- **AI引擎**：端侧ReAct Agent循环，30步最大推理深度，120秒长上下文，3次自动重试
- **工具层（13个ReAct工具）**：
  - **教务本地工具（4个）**：查课表、加活动、查空教室、添加课表安排
  - **后端知识工具（4个）**：课程搜索、课程详情、教师搜索、课程评价提交
  - **网络搜索工具（3个）**：网页搜索、AI即时回答（Bocha）、RAG知识库检索
  - **网页爬取工具（3个）**：单页爬取、整站爬取、爬取状态查询
- **后端服务**：
  - `AgentBackendClient`：课程/教师/RAG/搜索/爬取统一代理
  - `PrServerClient`：课程评价通过GitHub PR提交，社区共治
  - `MiniMax API`：大模型推理，支持HTTP 529/503/429自动退避重试
- **数据层**：
  - 三校区EAS适配器（深圳/本部/威海），自动识别校区路由
  - 智能缓存：空教室5周数据后台静默缓存，支持离线查询
  - 多轮对话历史持久化（Room数据库）

**视觉建议**：

- 工具层用图标矩阵展示13个工具，分4组配色

## 第7页：核心功能2 —— ReAct智能引擎：看得见思考过程

**标题**：Agent不只是给答案，还让你看见它是怎么想的

**要点**：
- **端侧ReAct循环**：Thought（思考）→ Action（调用工具）→ Observation（观察结果）→ 循环直到得出答案
- **深度推理**：最大支持30步工具调用，解决复杂多条件查询（如"找一间下周三下午第3-4节、容量大于50人的空教室"）
- **韧性设计**：
  - LLM API 3次自动重试，支持HTTP 529/503/429退避策略
  - 网络超时120秒，适配长上下文ReAct场景
  - 30步用尽时自动基于已有观察结果生成总结回答，不直接报错
- **思考可视化**：
  - 实时展示每一步的推理过程（"正在分析您的问题…" → "正在查询课表…" → "正在搜索空教室…"）
  - 保留最近30步思考历史，用户可查看完整推理链
  - 增强用户信任，降低"黑盒焦虑"

**演示脚本（配录屏）**：
1. 用户输入："帮我找找下周三我下课以后，能一直自习的教室"
2. Agent步骤1（Thought）：用户需要空教室，条件是下周三下课后
3. Agent步骤2（Action）：调用 `get_timetable` 确认下周三是否有课
4. Agent步骤3（Action）：调用 `search_empty_classroom` 查询空教室
5. Agent步骤4（Observation）：获得原始空教室列表，过滤时间
6. Agent最终回答：列出符合条件的教室及容量

**视觉建议**：
- 截图展示AI对话界面的"思考过程"气泡
- 配合流程图展示ReAct的5个步骤

## 第8页：核心功能3 —— 三校区空教室智能查询

**标题**：一校三区，一个Agent全覆盖

**要点**：
- **跨校区适配**：自动识别用户校区，调用对应教务系统接口
  - 深圳：`mjw.hitsz.edu.cn`，JSON API
  - 本部：`jwts.hit.edu.cn`，HTML表格解析
  - 威海：`webvpn.hitwh.edu.cn`，WebVPN穿透+HTML解析
- **智能缓存**：登录后后台自动缓存未来5周空教室数据，支持秒级离线查询
- **AI工具化**：将空教室查询封装为Agent工具，AI可自主决定何时调用
- **数据一致性**：解决不同校区数据格式差异
  - 本部/威海：存储全部84节次（空闲+占用），通过`PKBJ`标记占用状态
  - 深圳：仅存储占用节次
  - 工具层统一解析逻辑，对Agent透明

**技术亮点**：
- `awaitLiveData`协程同步机制，解决异步LiveData在后台线程的阻塞等待
- `MediatorLiveData`缓存优先策略：先读本地缓存立即展示，再挂网络请求刷新

**视觉建议**：
- 地图或校区标识 + 统一对话界面截图
- 缓存前后对比：左"加载中" → 右"秒出结果"
